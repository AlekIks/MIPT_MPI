Основная концепция игры "Жизнь" состоит в следующем
(правила игры доступны по ссылке (https://ru.wikipedia.org/wiki/Игра_«Жизнь»)):

* Есть бесконечная клетчатая доска (для аналогии, будем использовать тор размера `MxN`)
* В каждой клетке доски может находиться бактерия
* На каждом ходу, в зависимости от числа соседних клеток, в которых находится живая бактерия, либо появляется новая бактерия, либо умирает

----------------------------------

START <num rows> <num columns> <init status>

Параметры:
* num rows - количество строк
* num columns - количество столбцов
* init status - способ инициализации
	- random - случайная инициализация поля
	- csv - инициализация поля из файла с расширением (https://ru.wikipedia.org/wiki/CSV) (дополнительно необходимо в команде передать путь к csv-файлу)

Конечная цель команды - подготовить игру к запуску

----------------------------------

RUN <num steps>

Параметры:
* num steps - количество шагов

Должно запуститься вычисление игры на `num_steps` шагов. При этом команды могут добавляться много раз (то есть после ввода команды RUN можно вводить новые команды)

----------------------------------

STOP

Возвращаемые значения: 
* номер итерации игры, которая была при введении этой команды
* было ли прервано вычисление новой итерации или все шаги уже были выполнены, и процесс "простаивал"

----------------------------------

STATUS

Должен рисовать поле игры на экране. Работает только в том случае, если не производится просчет итераций (иначе нужно выводить соответствующее сообщение об этом)

----------------------------------

RESET

Команда должна аккуратно завершать вычисления (производить команду STOP) и очищать память. Цель команды: сделать после нее команду `START`

----------------------------------

QUIT

Завершить все процессы, связанные с игрой (STOP + RESET + завершение всех процессов)
